# Personal Study & Work Dashboard 开发流程文件

## 1. 项目目标

请开发一个个人学习与工作任务管理仪表盘网站，用于记录每日任务、学习进度、工作进度、复盘内容和阶段性总结。

这个项目的核心不是普通 Todo List，而是一个“个人成长看板”，需要有清晰的视觉层级、进度百分比、任务状态、复盘记录和数据统计。

## 2. 技术栈要求

优先使用：

- Next.js
- React
- TypeScript
- Tailwind CSS
- shadcn/ui
- lucide-react
- recharts
- localStorage 作为第一版数据存储

暂时不要接复杂后端。第一版先做纯前端可运行版本。

## 3. 页面结构

请开发以下页面或模块：

### 3.1 首页 Dashboard

展示今日整体状态：

- 今日日期
- 今日总进度百分比
- 学习任务完成率
- 工作任务完成率
- 今日重点任务
- 今日复盘入口
- 本周进度折线图
- 任务分类占比图

### 3.2 今日任务看板

任务字段包括：

- 任务名称
- 分类：学习 / 工作 / 毕业论文 / 作品集 / 英语 / 生活
- 优先级：高 / 中 / 低
- 状态：未开始 / 进行中 / 已完成 / 暂停
- 预计用时
- 实际用时
- 进度百分比
- 备注

功能要求：

- 新增任务
- 编辑任务
- 删除任务
- 修改状态
- 拖拽或点击切换任务状态
- 自动计算今日总进度

### 3.3 详细记录页

用于记录每个任务的过程：

- 今天做了什么
- 遇到的问题
- 解决方式
- 明天继续做什么
- 相关链接或备注

### 3.4 每日复盘页

包含以下输入区：

- 今日完成了什么
- 今日没有完成什么
- 原因是什么
- 今天最有价值的一件事
- 今天的情绪状态
- 明天最重要的三件事

复盘页面需要温和、简洁，不要像企业后台，要有个人陪伴感。

### 3.5 周/月统计页

展示：

- 本周任务完成数量
- 本周平均完成率
- 学习总时长
- 工作总时长
- 不同分类任务占比
- 连续记录天数
- 最近 7 天进度趋势

使用 recharts 做图表。

## 4. 视觉风格要求

整体风格：

- 高级、干净、轻量
- 类似 Notion + Apple Health + 数据仪表盘
- 背景不要太花
- 使用白色、浅灰、克莱因蓝作为主色
- 卡片使用圆角、轻阴影、毛玻璃效果
- 页面要有呼吸感，留白充足

主色建议：

- 克莱因蓝：#002FA7
- 背景色：#F7F8FA
- 文字主色：#111827
- 辅助灰色：#6B7280

## 5. 数据结构

请先使用 localStorage 保存数据。

Task 数据结构：

```ts
type Task = {
  id: string;
  title: string;
  category: "学习" | "工作" | "毕业论文" | "作品集" | "英语" | "生活";
  priority: "高" | "中" | "低";
  status: "未开始" | "进行中" | "已完成" | "暂停";
  estimatedMinutes: number;
  actualMinutes: number;
  progress: number;
  note: string;
  date: string;
  createdAt: string;
  updatedAt: string;
};
